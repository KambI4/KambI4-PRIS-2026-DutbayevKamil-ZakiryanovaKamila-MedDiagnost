# MedDiagnost

Учебная интеллектуальная система первичной медицинской оценки,
разрабатываемая в рамках дисциплины
**«Проектирование и разработка интеллектуальных систем»**.

Система используется исключительно в учебных целях
и не является медицинским инструментом.

---

# Курс: "Проектирование и разработка интеллектуальных систем"

## Видение проекта

**MedDiagnost** демонстрирует эволюцию интеллектуальной системы:

- от жесткой rule-based логики
- к структуре знаний на основе графа
- к обработке естественного языка и изображений
- к API и пользовательскому веб-интерфейсу

**Архитектура:** гибридная система, сочетающая:
- Символический ИИ: граф знаний, правила, rule-based inference
- Субсимволический ИИ: NLP через spaCy, Computer Vision через OpenCV/EasyOCR

```text

│   └─ Простуда

    ├─ Ибупрофен
    └─ Противовирусные
```

### Функциональность

```python
related_entities = find_related_entities(graph, "Кашель")
neighbors = graph.neighbors("Грипп")
```

### Результат
Система умеет искать связи между сущностями:
Симптом ↔ Болезнь ↔ Лекарство.

## Неделя 9-10: Интеграция модулей (Mid-term Review)

### Тема
Объединение rule-based логики и AI-компонентов.

### Архитектура unified pipeline

```text
Вход (текст / изображение)
        ↓
   DiagnosisPipeline
        ├─→ Текстовый путь
        │   ├─ process_text_message
        │   ├─ NER и извлечение сущностей
        │   ├─ Поиск по графу знаний
        │   ├─ Rule-based проверка
        │   └─ Оценка уверенности
        │
        └─→ Визуальный путь
            ├─ Классификация изображения
            ├─ OCR
            ├─ Извлечение медицинской информации
            └─ Диагностические подсказки
```

### Компоненты

| Файл | Назначение |
|------|-----------|
| `src/pipeline.py` | Unified pipeline для end-to-end сценариев |
| `src/services/diagnostics_service.py` | Сервисный слой и orchestration |
| `src/logic.py` | Текстовый анализ и правила |
| `src/vision.py` | Анализ изображений |
| `src/recommendation.py` | Рекомендательный модуль |
| `src/knowledge_graph.py` | База знаний |

### Результат
Проект объединяет символьный и субсимвольный подходы в единую учебную систему.
Для end-to-end сценариев предусмотрен `DiagnosisPipeline`, а пользовательские точки входа работают через `DiagnosticsService`.
Сервисный слой выдает не только ранжирование заболеваний, но и экспертный профиль: тяжесть, срочность, рекомендации и признаки для обращения за помощью.
