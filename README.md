# MedDiagnost 🩺

Учебная система первичной медицинской диагностики,
разрабатываемая в рамках дисциплины  
**«Проектирование и разработка интеллектуальных систем»**.

---

## 1. Описание проекта

**MedDiagnost** — учебный проект, демонстрирующий основные подходы
к построению интеллектуальных систем:

- продукционные модели (Rule-Based Logic);
- объектно-ориентированное моделирование (OOP);
- графы знаний (Knowledge Graph);
- диалоговый интерфейс (Chatbot).

Система не является медицинским инструментом и используется
исключительно в учебных целях.

В рамках первой лабораторной работы были выполнены следующие задачи:

## 3. Лабораторная работа №2  
**Продукционная модель и база знаний**

Во второй лабораторной работе реализована продукционная модель
принятия решений на основе правил вида **if–then**.

Основные компоненты:

- **Mock Data** — имитация данных пациента;
- **Inference Engine** — логика проверки правил;
- **rules.json** — база знаний с порогами и ограничениями;
- **Hard Filter** — критическая проверка (например, статус регистрации);
- интерактивный интерфейс Streamlit для тестирования сценариев.

---

## 4. Лабораторная работа №3  
**Объектная модель и граф знаний**

В третьей лабораторной работе система была расширена
за счёт объектно-ориентированной модели и графа знаний.

Реализовано:

- объектная модель медицинской предметной области
  (болезни, симптомы, лекарства) с использованием `dataclass`;
- граф знаний на базе библиотеки **NetworkX**;
- узлы графа: болезни, симптомы, лекарства;
- рёбра графа: медицинские связи между сущностями.

Граф знаний позволяет:

- по симптому определить связанные заболевания;
- по заболеванию увидеть симптомы и рекомендуемые лекарства;
- исследовать медицинский контекст через связи между объектами.

---

## 5. Лабораторная работа №4  
**Разработка диалогового интерфейса (Chatbot v1.0)**

В четвертой лабораторной работе реализован чат-бот с памятью переписки,
который ищет ответы в графе знаний.

Реализовано:

- чат-интерфейс на Streamlit с `st.chat_input` и `st.chat_message`;
- хранение истории диалога через `st.session_state.messages`;
- функция `process_text_message()` в `src/logic.py`;
- поиск терминов в графе знаний и вывод связанных сущностей;
- обработка русских/английских запросов и базовых синонимов;
- подсказки при опечатках и неизвестных терминах;
- ответ по критерию DoD: при отсутствии термина —
  **«Я не знаю такого термина»**.

Примеры запросов:

- `кашель`, `грипп`, `простуда`;
- `cough`, `flu`, `covid`;
- `болит голова`, `слабость и температура`.

## 6. Архитектура системы

```mermaid
graph TD;
    User[Пользователь] -->|Текстовый запрос| UI[Streamlit Chat Interface];
    UI -->|Сообщение| Logic[process_text_message()];
    Logic -->|Поиск по узлам и алиасам| Graph[Knowledge Graph (NetworkX)];
    Graph -->|Связанные сущности| Logic;
    Logic -->|Текстовый ответ| UI;
    UI -->|История диалога| Memory[st.session_state.messages];
```
