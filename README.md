## **Неделя 7: Компьютерное зрение (Computer Vision)**

### Тема
OCR и классификация изображений. Обработка визуального контента.

### Компоненты

| Файл | Назначение |
|------|-----------|
| `vision.py` | CV pipeline |

### Реализованные функции

#### 1. OCR (Оптическое распознавание символов)

```python
# EasyOCR - извлечение текста из изображения
reader = easyocr.Reader(["ru", "en"])
results = reader.readtext(image)
# → список текстовых блоков с координатами и уверенностью
```

#### 2. Предобработка изображения (OpenCV)

```python
# Улучшение контрастности для лучшего OCR
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
denoised = cv2.GaussianBlur(gray, (3, 3), 0)
threshold = cv2.adaptiveThreshold(denoised, 255, ...)
```

#### 3. Классификация медицинского изображения

**Типы:**
- `prescription_document` - медицинский рецепт
- `diagnostic_report` - диагностическое заключение
- `xray_image` - рентген
- `medical_document` - общий медицинский документ

**Признаки для классификации:**
- Средняя яркость пикселей
- Стандартное отклонение яркости
- Плотность границ (edges)
- Доля светлых пикселей
- Анализ OCR-текста (ключевые слова)

#### 4. Медицинская информация из OCR

```python
# Выделение диагнозов
pattern_diagnosis = r"диагноз|заключение|основной диагноз"

# Выделение лекарств
pattern_med = r"парацетамол|ибупрофен|амоксициллин"

# Выделение МКБ-кодов
pattern_icd = r"[A-Z]\d{2}(\.\d)?"
# → A10.0, E11.9, etc.

# Выделение назначений (препарат, доза, частота)
pattern_rx = r"(\w+)\s+(\d+\s*мг)\s+(.*?раз)"
```

### Примеры

**Входные данные:** Рентген грудной клетки (изображение)

**Обработка:**
```
1. Загрузка изображения
2. Классификация: "xray_image" (уверенность 0.95)
3. Визуальные признаки: 
   - mean_intensity: 82.0
   - edge_density: 0.19
4. OCR: "Рентген грудной клетки. Легкие без очаговых теней..."
5. Медицинское извлечение:
   - Диагноз: "Легкие без очаговых теней"
   - МКБ: нет
   - Назначения: нет
```

### Результат
Система обрабатывает визуальный контент (рецепты, рентгены, диагнозы)

## **Неделя 8: Гибридные методы и Рекомендации**

### Тема
Векторные представления и метрики близости (Cosine Similarity).

### Компоненты

| Файл | Назначение |
|------|-----------|
| `similarity.py` | Вычисление векторной близости |

### Реализованные методы

#### 1. Векторизация медицинских объектов

**Для изображений:**
```
Visual vector = [
    mean_intensity,
    std_intensity,
    edge_density,
    white_ratio,
    ocr_length
]
```

**Для текста:**
```
Text vector = TF-IDF представление
            медицинских терминов
```

#### 2. Вычисление Cosine Similarity

```python
similarity = (v1 · v2) / (||v1|| * ||v2||)
# Результат: 0.0 до 1.0
```

#### 3. Гибридная оценка

```
# Для рентгена:
final_score = 0.7 * visual_score + 0.3 * text_score

# Для рецепта:
final_score = 0.3 * visual_score + 0.7 * text_score

# Для текстового чата:
final_score = similarity(symptoms_vector, disease_vector)
```

### Примеры

**Сценарий 1: Рентген**
```
Пользователь загружает: рентген грудной клетки

Система находит похожие объекты в БД:
1. "Рентген пневмонии" - близость 0.92 (visual 0.95 + text 0.85)
2. "Рентген бронхита" - близость 0.78 (visual 0.81 + text 0.73)
3. "Рентген норма" - близость 0.45 (visual 0.48 + text 0.40)
```

**Сценарий 2: Текст**
```
Ввод: "кашель и температура"

Система рекомендует:
1. Грипп (0.94) - близость по симптомам
2. COVID-19 (0.87)
3. Простуда (0.81)
```

### Результат
Система находит похожие объекты и рекомендует решения на основе векторных представлений

## **Неделя 9-10: Интеграция модулей (Mid-term Review)**

### Тема
Объединение правил (символический) и NLP/CV (субсимволический).

### Архитектура unified pipeline

```
Вход (текст / изображение / структурированные данные)
        ↓
    Pipeline
        ├─→ [NLP Path] (logic.py)
        │   ├─ Нормализация текста
        │   ├─ NER + лемматизация
        │   ├─ Поиск в графе
        │   └─ Рекомендации
        │
        ├─→ [CV Path] (vision.py)
        │   ├─ Классификация
        │   ├─ OCR
        │   ├─ Извлечение информации
        │   └─ Рекомендации
        │
        └─→ [Rules Engine] (logic.py)
            ├─ Проверка критических симптомов
            ├─ Валидация температуры
            ├─ Статус пациента
            └─ Уровень безопасности
        ↓
Выход (DiagnosisResult с уверенностью)
```

### Компоненты

| Файл | Назначение |
|------|-----------|
| `pipeline.py` | DiagnosisPipeline - главный оркестратор |
| `logic.py` | Текстовый анализ + правила |
| `vision.py` | Анализ изображений |
| `similarity.py` | Рекомендации |
| `knowledge_graph.py` | База знаний |

### Класс DiagnosisPipeline

```python
class DiagnosisPipeline:
    
    def diagnose_from_text(text: str) -> DiagnosisResult
    # Путь: текст → NLP → граф → правила → результат
    
    def diagnose_from_image(image_bytes) -> Tuple[ImageAnalysisResult, Optional[DiagnosisResult]]
    # Путь: изображение → CV → OCR → (текст) → текстовый путь
    
    def diagnose_from_request(request: DiagnosisRequest) -> DiagnosisResult
    # Путь: структурированные данные → текстовый путь + проверка правил
```

### Структурированный результат (dataclass)

```python
@dataclass
class DiagnosisResult:
    primary_candidates: List[Dict]      # Основные диагнозы
    related_symptoms: List[str]         # Связанные симптомы
    recommended_medicines: List[str]    # Рекомендуемые лекарства
    confidence: DiagnosisConfidence     # HIGH / MEDIUM / LOW
    rules_check: str                    # Статус безопасности
    reasoning: str                      # Пошаговое объяснение
    entities_found: List[tuple]         # Извлеченные сущности (NER)
```

### Примеры использования

```python
# Пример 1: Текстовый запрос
pipeline = DiagnosisPipeline()
result = pipeline.diagnose_from_text("кашель и температура")
# → DiagnosisResult с диагнозами, симптомами, лекарствами

# Пример 2: Анализ рецепта
with open('recipe.jpg', 'rb') as f:
    img_result, diag = pipeline.diagnose_from_image(f.read())
# → ImageAnalysisResult + DiagnosisResult

# Пример 3: Структурированные данные
req = DiagnosisRequest(
    patient_symptoms=["кашель", "температура"],
    temperature=38.5,
    is_registered=True
)
result = pipeline.diagnose_from_request(req)
# → DiagnosisResult с проверкой правил
```

### Результат
**Единый интерфейс** для всех типов входных данных
**Mid-term Review** - система интегрирует все компоненты

### Архитектура

```
┌─────────────────────────┐
│   Streamlit UI          │ ← Представление (Presentation)
│   (main.py)             │
└────────┬────────────────┘
         │ вызывает функции
         ▼
┌─────────────────────────┐
│   DiagnosisPipeline     │ ← Бизнес-логика (Business Logic)
│   (pipeline.py)         │
└────────┬────────────────┘
         │ использует
         ▼
┌─────────────────────────┐
│   Data Processing       │ ← Обработка данных
│   logic, vision,        │
│   similarity            │
└────────┬────────────────┘
         │ читает
         ▼
┌─────────────────────────┐
│   Knowledge Base        │ ← Данные
│   Graph, Rules, JSON    │
└─────────────────────────┘
```

### Модульная структура проекта

```
src/
├── pipeline.py              ← Unified Pipeline (главный класс)
├── main.py                  ← Streamlit UI (только представление)
├── logic.py                 ← Текстовая обработка (NLP)
├── vision.py                ← Компьютерное зрение (CV + OCR)
├── similarity.py            ← Векторные рекомендации
├── knowledge_graph.py       ← Управление графом знаний
├── models.py                ← Структуры данных
└── mock_data.py             ← Тестовые данные
```

### Разделение ответственности

| Модуль | Ответственность |
|--------|-----------------|
| `pipeline.py` | Оркестрирует все компоненты (NLP + CV + Rules) |
| `main.py` | Только UI/UX через Streamlit |
| `logic.py` | NLP: нормализация, NER, поиск, граф |
| `vision.py` | CV: классификация, OCR, извлечение |
| `similarity.py` | Вычисления близости и рекомендации |
| `knowledge_graph.py` | CRUD операции над графом|
| `models.py` | Структуры данных (dataclasses, enums) |
